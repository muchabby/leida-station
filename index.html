📘 搭建 SOP · 一起学习

从前期建站到上云、接入飞书的完整流程，欢迎大家照着复刻

查看完整 SOP 文档 →
阶段一
前期网页建设

定品牌词与监测对象 → 搭纯前端单页 index.html → 写无依赖抓取脚本 fetch.js → 约定 data.js 数据格式 → 本地试跑调噪音。

阶段二
本地定时跑

上云前的过渡：Windows 任务计划每天 9/13/18 点跑，9 点抓全套含社交，SerpAPI Key 存环境变量。

阶段三
上云（GitHub）

建公开仓库 + 开 Pages 托管 → 密钥存 GitHub Secrets → Actions 每天 9 点定时抓取、bot 自动 commit data.js，完全不依赖本地电脑。

阶段四
多设备星标同步

Supabase 存 user_marks 表，前端用 publishable key + RLS 保护；启动先拉云端再渲染，避免多设备竞态。

阶段五
接入飞书推送

自定义机器人 + 关键词「雷达站」安全设置；fetch.js 的 pushLark() 负面置顶、有新增才推。

阶段六
日常运维与扩展

盯紧 SerpAPI 额度（90 次/月）、加词改频率的维护动作、密钥存放总表与已知局限。
